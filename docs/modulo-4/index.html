MÓDULO 4
Desenvolvimento, Qualidade e Versionamento de Software

Curso:
Formação em Processo de Desenvolvimento de Software (PDS) e DevSecOps

Carga Horária:
6 horas (3h EaD + 3h Presencial)
Objetivo do Módulo

Capacitar os participantes para a execução do desenvolvimento de software de forma padronizada, controlada e orientada à qualidade, conforme o PDS institucional, abordando boas práticas de codificação, controle de versões, revisão técnica e preparação do código para automação em esteira DevSecOps.

1. Desenvolvimento de Software como Atividade Institucional

No contexto institucional, o desenvolvimento de software não é uma atividade individual ou artesanal, mas um processo organizacional crítico, sujeito a regras, controles, auditorias e responsabilidades formais. Cada linha de código produzida representa uma decisão técnica que pode impactar a segurança, a continuidade dos serviços públicos e a confiabilidade das informações tratadas.

Diferentemente de ambientes informais, no setor público o código-fonte é um ativo institucional, devendo ser rastreável, versionado, auditável e alinhado às diretrizes de governança estabelecidas no PDS.

Nota Código sem padrão, sem histórico e sem revisão não é apenas um risco técnico, mas um risco institucional.

2. Padrões de Codificação e Qualidade

A padronização da codificação é um dos pilares da qualidade de software. O PDS estabelece que os projetos devem adotar padrões claros de nomenclatura, organização de diretórios, estilo de código e documentação mínima, garantindo legibilidade, manutenibilidade e continuidade administrativa.

Esses padrões reduzem a dependência de indivíduos específicos, facilitam a entrada de novos desenvolvedores e permitem que ferramentas automatizadas analisem o código de forma consistente.

Leitura orientada Guia de Boas Práticas de Codificação adotado pela SEMA/MT.
Material complementar Clean Code – Princípios fundamentais de qualidade de código (SonarSource)
https://docs.sonarsource.com/sonarqube/latest/user-guide/clean-code/
Material externo SonarSource – Code Quality Concepts

3. Qualidade de Software no PDS

Qualidade de software não se resume à ausência de erros visíveis. No PDS, a qualidade é entendida como o atendimento simultâneo a requisitos funcionais, não funcionais, de segurança, desempenho, usabilidade e conformidade normativa.

Para isso, o processo prevê validações técnicas, testes, revisões e registros formais, que futuramente serão automatizados pela esteira DevSecOps.

Material complementar ISO/IEC 25010 – Systems and Software Quality Models (versão atual)
https://www.iso.org/standard/78176.html
Material externo ISO – Página oficial da norma ISO/IEC 25010
https://www.iso.org/standard/35733.html
Atenção Sem critérios claros de qualidade, a automação apenas acelera a entrega de software defeituoso.

4. Versionamento de Código-Fonte

O versionamento é um mecanismo essencial de controle, rastreabilidade e governança. Todo código deve ser mantido em repositórios institucionais, com histórico completo de alterações, responsáveis e justificativas.

O uso adequado de branches, commits pequenos e mensagens claras permite compreender a evolução do sistema e apoiar auditorias técnicas e administrativas.

Elemento Finalidade
Repositório Armazenamento centralizado e controlado do código
Commit Registro atômico de alterações
Branch Isolamento de desenvolvimento e correções
Tag Identificação de versões e releases
Material complementar Git Book – Pro Git (Capítulos iniciais)
https://git-scm.com/book/en/v2
Material externo Git SCM – Reference Documentation
https://git-scm.com/docs

5. Revisão de Código (Code Review)

A revisão de código é uma prática obrigatória para garantir qualidade, segurança e conformidade. Ela permite identificar falhas, inconsistências, riscos de segurança e desvios de padrão antes da integração ao código principal.

No contexto do PDS, a revisão também cumpre papel educativo e de disseminação de boas práticas entre a equipe.

Material complementar Google Engineering Practices – Code Review Guidelines
https://google.github.io/eng-practices/review/
Material externo GitLab – Code Review Guidelines
https://docs.gitlab.com/ee/development/code_review.html
Resumo conceitual Código revisado → qualidade, segurança e aprendizado
Código sem revisão → risco técnico e institucional

6. Preparação do Código para a Esteira DevSecOps

O desenvolvimento deve ser realizado considerando, desde o início, a futura automação em pipelines CI/CD. Isso implica código modular, testes automatizáveis, configurações externas e ausência de dependências rígidas de ambiente.

Essa preparação é fundamental para viabilizar práticas como integração contínua, análise automática de qualidade e segurança e deploy controlado.

Material externo OWASP Top 10 – Principais Riscos de Segurança em Aplicações Web
https://owasp.org/www-project-top-ten/
Material externo NIST Secure Software Development Framework (SSDF)
https://csrc.nist.gov/projects/ssdf
Encadeamento técnico Desenvolvimento padronizado → versionamento → revisão → automação → entrega segura

7. Rastreabilidade e Auditoria

Cada alteração no software deve ser rastreável a uma demanda, requisito ou correção. Essa rastreabilidade é essencial para auditorias, investigações de incidentes e prestação de contas institucional.

Ferramentas de versionamento e gestão de demandas atuam conjuntamente para garantir essa transparência.

8. Síntese Final

Ao final deste módulo, o participante deverá compreender que desenvolvimento, qualidade e versionamento são dimensões inseparáveis do PDS. O código produzido deve estar preparado não apenas para funcionar, mas para ser mantido, auditado, automatizado e evoluído com segurança no contexto da esteira DevSecOps.
